# Luffi — Умное бронирование помещений (Демо-прототип)

Этот проект представляет собой полностью рабочий фронтенд-прототип системы умного бронирования переговорных комнат. Он создан на чистом HTML, CSS и JavaScript и демонстрирует весь цикл работы: от поиска свободной комнаты до автоматического контроля доступа и аналитики.

## Основной функционал

- **Просмотр и бронирование комнат**: Пользователь видит все доступные переговорные, их статус (свободна/забронирована/занята) и вместимость. Бронирование выполняется в один клик.
- **Генерация цифрового ключа**: При успешном бронировании система генерирует уникальный QR-код и цифровой ключ, который служит пропуском в комнату.
- **Автоматическая отмена "комнат-призраков"**: Если в течение 15 секунд (в демо-режиме) после бронирования никто не "вошёл" в комнату, бронь автоматически отменяется, а комната снова становится доступной. Это имитирует работу датчиков присутствия.
- **Динамическая аналитика**: Интерфейс в реальном времени отображает:
    - Список активных броней.
    - Процент загрузки каждой комнаты.
    - Общий процент пустующих помещений.
    - Количество автоматически отменённых броней ("призраков").
- **Таймер обратного отсчета**: Для активной брони отображается оставшееся время использования комнаты.

## Технологии

Проект использует только клиентские технологии и не требует серверной части для работы.

- **HTML5** — Семантическая разметка страницы.
- **CSS3** — Кастомные стили, адаптивная вёрстка (Flexbox/Grid), анимации.
- **JavaScript (ES6+)** — Вся логика приложения: управление состоянием, рендеринг интерфейса, обработка событий, таймеры.
- **QRCode.js** — Внешняя библиотека для генерации QR-кодов прямо в браузере.

## Структура кода

### 1. Данные
Вся информация о комнатах и бронированиях хранится в JavaScript-массивах (`roomsData`, `bookings`). Это позволяет быстро изменять состояние и перерисовывать интерфейс.

### 2. Основные функции
- `bookRoom(roomId)` — Создаёт новую бронь, обновляет статус комнаты, генерирует QR-код и запускает таймер.
- `checkGhost(roomId)` — Проверяет, была ли активирована бронь (в демо-версии — просто отменяет её через 15 секунд), и освобождает комнату.
- `renderAll()` — Главная функция рендеринга, которая обновляет все части интерфейса: список комнат, активные брони и аналитику.

### 3. Логика работы
1.  Пользователь нажимает кнопку "Забронировать" на свободной комнате.
2.  Система создаёт бронь на 1 час, генерирует QR-код и ключ доступа.
3.  Запускается таймер обратного отсчёта.
4.  Через 15 секунд, если бронь не была "активирована" (в демо-версии это происходит автоматически), срабатывает защита от "комнат-призраков": бронь отменяется, и комната освобождается.
5.  Все изменения мгновенно отображаются в интерфейсе благодаря функции `renderAll()`.

Для запуска проекта достаточно открыть файл `index.html` в любом современном браузере. Все зависимости (библиотека QRCode.js) подгружаются через CDN.

# Клонируйте репозиторий (если применимо)
git clone <ссылка-на-репозиторий>

# Откройте файл в браузере
open index.html
